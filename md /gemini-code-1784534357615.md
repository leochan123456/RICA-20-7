# Role & Task
You are an expert Python/TypeScript Full-Stack Engineer. Build a lightweight automated service/script that monitors social media platform policy updates and maintains a dynamic `restriction_rules.json` database for an AI posting agent (Rica+ GPA).

# Core Requirements & Workflow

1. **Policy Scraping & Monitoring (Telemetry Module)**:
   - Create a scraper using `playwright` or `httpx` to check official developer/policy URLs or RSS feeds for platforms:
     - Meta (Facebook/Instagram HEC & Community Standards)
     - Xiaohongshu (RED Community Guidelines & Advertising Law)
     - Douyin (Juhaofang / Real Estate Terms)
   - Store raw HTML/text snapshots locally under `./snapshots/` with timestamps.

2. **Diff & Keyword Extraction Engine**:
   - Compare the newly fetched page content with the previous snapshot using `difflib` or an LLM API call.
   - Detect new banned/sensitive keywords (e.g., PRC Advertising Law triggers like "学区房", "保证收益").

3. **Database Auto-Updater (`restriction_rules.json`)**:
   - Parse changes and automatically update or append rules into a structured JSON file:
     ```json
     {
       "platform": "Xiaohongshu",
       "last_updated": "2026-07-20",
       "banned_keywords": ["学区房", "保证收益"],
       "link_in_caption_allowed": false,
       "recommended_action": "replace_with_proximity_term"
     }
     ```

4. **Shadowban Self-Checking Mock API**:
   - Implement a function `check_shadowban_status(post_url: str, platform: str) -> dict` that checks if post views/reach are abnormally suppressed (views == 0 after 2 hours) and logs flags to SQLite or JSON.

# Instructions for Copilot
- Write clean, modular, and fully commented code.
- Use Python with `asyncio`, `playwright`/`httpx`, `pydantic` for data validation, and `rich` for CLI logging.
- Structure project into:
  - `scraper.py` (Fetching policy pages)
  - `analyzer.py` (Diff & keyword parsing)
  - `db_manager.py` (JSON/SQLite CRUD)
  - `main.py` (Execution entry point & cron schedule)
